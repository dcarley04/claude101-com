## Repository Overview

This repository contains LinkedIn post templates and content generation tools. The primary purpose is to create consistent, professional LinkedIn content focused on AI agent projects and technical achievements.

## File Structure

The repository currently contains:
- `post1.md`, `post2.md`, `post3.md`: Sample LinkedIn post templates
- Standard Git files (.git)

## Working with Content

### Content Creation Process

1. Posts generally follow a template structure:
   - Opening statement about completed project/course
   - Mention of specific tools/platforms used
   - Benefits and ease of use statements
   - Recognition of specific people/teams
   - Bullet points explaining functionality
   - Relevant hashtags

2. Posts typically highlight AI agent projects built with platforms like Relay.app

3. Tone is professional yet enthusiastic, focusing on completed achievements

### Style Guidelines

- Use clear, jargon-free language accessible to professionals
- Maintain consistent formatting with numbered lists using "1/" format
- Include 3-5 relevant hashtags at the end of each post
- Always thank specific individuals/teams when mentioning tools/platforms

## Development Commands

Since this is a content-focused repository rather than a codebase, typical development commands aren't applicable. However, standard Git workflows should be used:

```bash
# Check status
git status

# Add changes
git add .

# Commit changes with descriptive message
git commit -m "Description of changes"

# Push to GitHub
git push origin main
```

### Version Control Practices

All work should be committed to Git and pushed to GitHub regularly with clean, descriptive commit messages. This ensures:

1. Work is never lost if there are system issues
2. Progress can be tracked and rolled back if needed
3. Changes are properly documented with meaningful commit messages
4. The latest work is always available on GitHub for backup

Commit frequently during work sessions and always push changes before stopping work. Each commit should represent a logical unit of work with a clear, descriptive message.

## Architecture Notes

This is a simple content repository without complex architecture. Files are standalone markdown documents that follow a consistent style pattern.

When creating new content:
